windows 下环境变量的读取顺序

Veröffentlicht am 2019-04-16

经测试： ### 算了，测试部分直接跳过，直接上结论

直接上结论：

第一：windows系统中，在查找path变量时，先查找系统环境变量，再查找用户环境变量

第二：windows 环境变量的查找顺序是按照文件的录入的顺序从前向后找的，最前面一条的优先级最高，如果找到命令就不会再向后找

ps1：因此新增环境变量的时候，最好将新添加的放在最前面 以获得最高优先级
ps2:另外：环境变量修改后需要关闭命令后后再打开才生效
